## 2. CleanShot X — the "as-used" walkthrough

### 2.0 The official pitch order (reveals intended hero features)
[OBSERVED] The 2020 official trailer (FZbICrBKWIU) demos features in this exact order: **(1) capture → smart overlay pops in corner → (2) annotate → (3) drag-and-drop to share → (4) Cloud upload = instant link → (5) scrolling capture → (6) screen recording / GIF → (7) auto-mute notifications while recording → (8) trim in app → (9) hide desktop icons + custom wallpaper → (10) single-window capture beauty → (11) pin to screen → (12) "hundreds of options."** Closing line: *"CleanShot X is not just an app, it's a complete workflow."* — the overlay-as-hub IS the product thesis.

### 2.1 First-run / onboarding flow ([OBSERVED] official "First steps", Feb 2022)
1. DMG → drag to Applications → launch.
2. License agreement → activate via **license key or CleanShot Cloud account** (online activation).
3. Setup wizard explicitly offers to **take over the macOS default shortcuts** (⌘⇧3/⌘⇧4) — but the user must manually go to System Preferences → Keyboard → Shortcuts and *uncheck Apple's defaults themselves* (the app deep-links there). This is a known onboarding speed bump in every walkthrough.
4. Wizard asks: "Would you like to hide desktop icons when you take a screenshot?" (presented as a flagship toggle at onboarding).
5. Prompt to sign in / create a free CleanShot Cloud account (can skip).
6. macOS Screen Recording permission prompt before first capture.
- Muscle-memory continuity is a selling point reviewers echo: *"if you have 20 years of mac os 10 experience taking screenshots you don't have to change your muscle memory"* (A Better Computer).

### 2.2 The core loop: capture → Quick Access Overlay (the hub)
Every single reviewer's demo passes through the same pivot point: after ANY capture, a thumbnail card ("Quick Access Overlay") slides into the bottom-left corner and **stays there until acted on** (configurable auto-close, default ~30s; multiple captures stack).
From the overlay you can (each with its own hotkey while hovering, e.g. ⌘E = edit):
- **Copy** to clipboard (and dismiss)
- **Save** to configured folder
- **Pin** — floats above all windows
- **Cloud upload** — link auto-copied to clipboard
- **Edit/Annotate** — opens the editor
- **Drag the thumbnail itself** into Slack/email/Pages/Google Docs/Finder — it behaves as a file
- **Close** (dismiss without saving — "the screenshot isn't saved anywhere yet")
Key behavioral detail [DEMO, multiple]: nothing is written to disk until you choose — *"I like having this version because it doesn't waste any storage space"* (Demetri Panici). The "After capture" settings let you re-wire the default (e.g., always copy to clipboard + show overlay; recordings → open editor directly and skip overlay). Sara Dietschy demos the **capture history** recall (⇧⌥X in her config): *"even if you swipe them away, they're not going to disappear… boom, you have all of your screenshots on essentially your clipboard."*

**Flow reconstruction — the most common reviewer workflow (Matt @ official video states it too):** capture area → overlay appears → ⌘E annotate → arrow/box/blur → Done (back to overlay with edits applied) → drag thumbnail into Slack/email. Total: one capture, zero file management.

### 2.3 Annotation editor flow
Tools demoed (in rough frequency order across videos): arrow (curvable), rectangle/box, filled rectangle, ellipse, line, text, **blur**, **pixelate**, **highlighter** ("snaps to text" — Sara Dietschy: *"highlight text and it keeps it on the text, which is very nice"*), **counter/numbering tool (1,2,3 or A,B,C)** for tutorials, pen/freehand, spotlight (darkens everything except selection), crop (snaps to lines; ⌘ to disable snapping; mirror/rotate promoted to top level in 4.8).
Interaction details reviewers call out:
- Option-drag duplicates an annotation object (A Better Computer).
- Objects stay editable/movable after placement: *"you can always select it and edit it like right here I can Boop move this Arrow somewhere else — it's much easier than any other screenshot product I've ever experienced"* (Demetri Panici).
- Color persists to next object drawn; 4.8 added a custom color picker with eyedropper + 10 saved swatches (official 4.8 video: pick the exact button-blue from the screenshot itself, save it).
- Can paste/drag *additional screenshots into the canvas* and snap-align them into grids (4.8).
- Misplaced counter? *"no problem just grab it and choose where you want it to go"* (Dave Swift).

### 2.4 The Background tool (the "make it pretty" flow — added v4.5, Dec 2022)
[DEMO — A Better Computer 4.5 video; Demetri; SaaS Report; Sara Dietschy] In the editor, press **B** → choose built-in gradients / own wallpaper / blurred wallpaper / solid color → adjust **padding**, **inset**, **corner radius**, **shadow**, **aspect-ratio lock (16:9, 3:2…)**, **auto-balance margins**. Dave Swift builds an entire **YouTube thumbnail** in it: *"I think that's a pretty nice looking YouTube thumbnail for about 30 seconds of work."* Sara Dietschy moved her screenshot beautification out of DaVinci Resolve into CleanShot. Windowed screenshots get the wallpaper composited behind rounded corners automatically; since 4.8 the background remains editable after the fact ("you're not locked into your initial choice").

### 2.5 Scrolling capture flow
Menu bar (or hotkey) → drag-select the region → "Start Capture" → **you scroll manually** (live preview pane builds on the right; app says *"please slow down"* if too fast) **or hit Auto-scroll** → Done → overlay/editor. 4.8 added **horizontal** scrolling capture (demoed on a Final Cut timeline). Auto-scroll pauses when mouse leaves the capture area (SaaS Report). Works in any app, not just browsers: *"as long as I can scroll it I can capture it"* (A Better Computer).

### 2.6 OCR flow
"Capture Text" from menu/hotkey → same crosshair → select → **text lands on clipboard instantly** (notification). Demoed on: text inside a *playing GIF* (A Better Computer — crowd-pleaser moment), tweets, non-selectable web text, paused videos. Option to keep/strip line breaks. Reviewers stress on-device privacy ("done completely on device, so there's no privacy concerns" — Serge Morgotch).

### 2.7 Screen recording flow (the "Loom replacement" use)
Record Screen → drag handles to select region (aspect-ratio lock, exact-pixel entry, spacebar = pick a window) → toolbar toggles: mic (incl. iPhone via Continuity), **system/internal audio**, webcam overlay (circle/square/vertical; click to swap PiP ↔ fullscreen *while recording*), show mouse clicks, show keystrokes → Record (video or GIF).
During recording: floating widget with **pause/resume** (Dave Swift: *"the recording picks up exactly where it left off… I don't know of another tool that does that"*), **restart** (*"so I don't end up with a string of bad takes"*), trash (with confirm).
After stop: per-settings, overlay or editor; editor = trim, resize/downscale, quality + live estimated file size, volume boost/mute, export MP4/GIF; retina = effectively 4K, FPS settable to 60.
Auto-upload pattern [DEMO MinorCo]: recordings auto-upload to Cloud, **link is copied before upload even finishes** — *"I can paste that into my email, send off my email and my video could still be uploading."*

### 2.8 Cloud sharing flow
Overlay → cloud icon → uploads → notification "link copied." Open link = hosted page with the image/video, download button, view counter. Extras demoed: file naming on upload, tags, **password protection**, **self-destruct timers** (1 day / 7 days / 30 days), custom domain (share.yourdomain.com), portal branding (logo/avatar), toggle download button off (Dave Swift). 1 GB free with license; Pro = unlimited + custom domain + branding removal.

### 2.9 Pin flow
Overlay → pin icon → screenshot floats above ALL windows at chosen position/size; used as reference while working ("grab a chart, keep taking notes"); also "Pin from file." Combined with OCR in Dave Swift's note-taking demo.

### 2.10 Utility features reviewers actually use outside capture
- **Hide desktop icons** via hotkey, used live during Zoom/Meet screen-shares: *"I'm not using clean shot x right now at all… but my desktop looks really nice"* (A Better Computer). Appears in nearly every review.
- Auto-DND while recording (kills notification embarrassment).
- Everything hotkey-able ("basically everything CleanShot X can do can be set to a keyboard shortcut").
- Open any external image/video via right-click → "Open with CleanShot" → annotate/upload (works as a general markup + share tool).
- Capture history restore ("restore recently closed items").
- File formats: PNG/JPEG/WEBP (4.8), HEIC open-only; sRGB color-management toggle.

## 3. Shottr — the "as-used" walkthrough

### 3.1 Install & first-run
Download DMG from shottr.cc → drag to Applications → Gatekeeper "open?" → menu bar icon appears. First-run invites you to **set up hotkeys**; to steal ⌘⇧3/⌘⇧4 the user must manually uncheck Apple's defaults in System Settings (same speed bump as CleanShot; ScreenCastsONLINE and Appychip both walk through it on camera). Default convention in most videos: **⌘⇧1 fullscreen, ⌘⇧2 area** ("you invoke it basically the same way you would invoke the regular screenshot tool" — Snazzy Labs). Separate Accessibility permission needed for scrolling capture (appsntips shows the double permission dance: *"if that felt too much, don't worry, you won't have to grant these permissions ever again"*).

### 3.2 The core loop: capture → preview window
After capture, a **preview window** appears (not a corner overlay like CleanShot — more of a floating editor-lite):
- Happy with it → **drag it to desktop/Finder** to save there, or hit **Copy** (copies + closes WITHOUT saving a file: *"if I know that I'll be deleting the screenshot instantly after I send it to someone…"* — Bog), or ⌘C immediately.
- Not happy → **Escape** to throw it away and retake (Bog contrasts: in Apple's tool "you have to click on the screenshot first and then hit the trash icon").
- **Enter** → opens the full editor.
- Drag-handle icon to drag the file anywhere (also closes it after drop).
- **Pin** to turn the shot into a floating always-on-top window (AppGazing).
- Reopen-last-screenshot **preserves all edits** — *"that's actually happened to me a few times and it's much more convenient than hunting down the file"* (AppGazing).
The whole loop is keyboard-first and transient — Shottr's feel is "throwaway-fast."

### 3.3 The editor — keyboard-driven, designer-flavored
Snazzy Labs (2026 video) rattles the whole flow off by single keys, which is itself the point: **B** blur, **H** highlighter, **C** counter, **M** magnifier (choose placement + zoom of a loupe inset), **K** backdrop (wallpaper/gradient/solid), **1**/**2** horizontal/vertical pixel rulers, **A** arrow, **T** text (with "dingleberry" pointer to make speech bubbles that "intelligently move around"), **V** selection → **Enter crops**, **Tab** copies hovered color's hex, **⌘C** copy-and-dismiss, drag-out from the viewer.
Standout tools every Shottr reviewer demos:
- **Smart blur/erase**: blur a region, **blur ONLY the text**, **erase ONLY the text**, or content-aware erase the whole object — across inconsistent backgrounds. This is THE Shottr wow moment (see §5).
- **Pixel rulers + point/pixel toggle**: *"it's the best measuring tool in mother-friggin macOS"* (Snazzy Labs 2026). Hold 1 / 2 to measure, click to **stamp the measurement onto the screenshot** (Jeff Su).
- **Hex color picking**: hover any pixel, current color shown in header, Tab copies hex (replaced a dedicated color-picker app for Jeff Su; Bog pastes straight into Photoshop).
- **AI object selection**: ⌘-click auto-selects an element; option-drag grows a smart selection; select two icons + Delete = content-aware removal ("you would never know" — Snazzy Labs).
- **Auto-adjust crop** ("tries to select exactly what I wanted").
- **Zoom/pan navigation**: scroll to zoom, hold right mouse to pan, ⌘2 to dive into a selection (Jeff Su).
- **Paste image into the canvas** (overlay a second image; AppGazing's recipe demo).
- OCR from inside the editor: click the "A" icon → all text in shot → clipboard (Bog).

### 3.4 Scrolling capture
Hotkey → crosshair → select area → **it auto-scrolls by itself** (don't touch mouse/keyboard; Esc to stop) → long shot opens in editor. Demoed on Safari, an RSS app (GoodLinks), chats. A Better Computer: "I'm not doing anything, it's just doing it on its own… a very high-res image, everything looks really nice."

### 3.5 OCR + QR flow
Hotkey (e.g. ⌘⌥⌃O or user-set ⌘⇧5) → crosshair → select → text on clipboard. Works on handwriting (AppGazing), QR codes → link straight to clipboard (Jeff Su uses it instead of pointing a phone at presentation slides). Pro tip culture: set primary OCR language in Advanced prefs for CJK accuracy (Jeff Su demos Chinese recognition failing under Latin default). Accuracy not perfect: Snazzy Labs gets "a weird at symbol"; Appychip's demo turns "Ming" into "non name" — both shrug it off.

### 3.6 What Shottr does NOT do (named on camera)
No video/screen recording, no signature tool, weak text styling ("not as great at modifying the text like changing the font or making the font size really big") — AppGazing. No built-in easy cloud: upload requires **emailing the developer for a token** (A Better Computer: "you need to get a token by emailing them — haven't done this"). Jeff Su's disclaimer: *"I still find myself using CleanShot X a lot more since I need to upload my screenshots to the cloud for easy sharing and it's slightly more responsive than Shottr."* (Note: "more responsive" claim is contested — most others call Shottr the lighter/faster one.)

### 3.7 Pricing-in-use (how it feels in videos)
Free forever with a gentle nag after 30 days — ScreenCastsONLINE: *"It's a very gentle nag… the developer refers to it as the annoying fellow."* $8 basic (she told the developer "I think he has a price too low"), $25 "friends club" (bragging rights, experimental features, better support). Snazzy Labs 2026 frames it as: *"you know how WinRAR is not free software but no Windows user has ever paid for WinRAR? Well there is a macOS equivalent called Shottr. Technically it's $12 but you can use it basically indefinitely with almost every feature for free… but honestly, pay 12 bucks for it."*

## 6. Friction moments (named on camera — the cut/fix list)

### CleanShot X
- **Manual shortcut surrender:** onboarding makes YOU go uncheck Apple's shortcuts in System Settings (official video walks it; every tutorial repeats it). Nobody automates it (macOS limitation, but it's the first 2 minutes of every user's life).
- **No free trial:** *"there's no free trial — you can buy it and there's a 30-day money-back guarantee"* (A Better Computer). Repeated purchase-anxiety theme; reviewers compensate by pointing to Setapp's 7-day trial.
- **Price + device licensing:** *"it's like 30-40 bucks or something — it's not nothing… if you have two computers, it's gonna disable one if you try to activate it on the other"* (A Better Computer). $29 = 1 device, +$19 for updates after year one (The Inventar, 2026, confirms $29 + $19/yr updates; $8–10/mo for Cloud Pro). Demetri Panici openly coaches license-reset workarounds and "don't buy the cloud thing": *"you do not need to waste $8 a month on this product"* — i.e., the Cloud subscription's value is contested even by superfans who route exports to Google Drive instead.
- **Cloud Pro upsell ambiguity:** Matt Birchler on camera, squinting at the pricing page: *"there is a pro tier that gets you some other features, um, not exactly sure what's all in there."*
- **Manual-scroll capture can outrun the stitcher** ("please slow down" warning; auto-scroll fixed this but several reviewers started with manual).
- **Background auto-extend (inset) only looks clean on solid-color shots:** *"sometimes… it'll do its best but it won't look this clean"* (SaaS Report).
- **Video editor is shallow by design:** *"the editor isn't super full featured"* (Dave Swift); for polished zooms reviewers bolt on Screen Studio (Steve Builds Websites: *"can clean shot record itself? No"*).
- OCR line-break handling needs a settings dig; stray characters from logos get captured (Dave Swift).

### Shottr
- **Notification-permission nag hack:** *"I must enable notifications for this app in order for it to stop giving me annoying pop-ups after every screenshot — but a workaround is to enable the notifications then select None"* (Bog). Real first-day jank.
- **Tab (hex copy) doubles as 'close without saving' depending on focus** — A Better Computer loses his edited image on camera: *"it did copy the color but I lost the image, so that's kind of an annoying implementation."*
- **Cloud upload requires emailing the dev for a token** (A Better Computer) — sharing is effectively clipboard/drag only.
- **No recording, no signature, weak text styling** (AppGazing) — acknowledged trade-offs.
- **Double permission prompts** (Screen Recording + Accessibility for scroll capture) with a quit-and-reopen step (appsntips).
- **"Slightly less responsive than CleanShot"** [CLAIM, contested] (Jeff Su).
- OCR accuracy hiccups (typos in output, on camera twice) and language setting buried in Advanced.

## 8. Cross-cutting flow insights for our product

1. **The post-capture hub is the product.** Both apps' entire feel comes from what happens in the 3 seconds after capture. CleanShot = persistent corner overlay with copy/save/pin/cloud/edit/drag; Shottr = instant editor-preview with Esc/Enter/⌘C/drag. Reviewers narrate THIS moment in every video; the capture itself is commodity. Our v1 must nail one of these two post-capture models (or a hybrid: overlay that expands into a keyboard-first editor).
2. **"Nothing saved until I decide" is loved on both sides.** Clipboard-only flows, no disk litter, drag-as-file. (Demetri, Bog, official.)
3. **Muscle-memory takeover of ⌘⇧3/⌘⇧4 is table stakes** — and the manual System Settings dance is a universally tolerated but ugly first-run step. Smoothing it is a visible differentiator in every onboarding video.
4. **Hero demos to win reviewers** (what makes them gasp on camera): scrolling capture that "just works" anywhere; text-aware blur/erase ("Next Level AI" — and a natural fit for our AI premium tier); background/beautify with padding+gradient ("social-ready in 30 seconds"); OCR from impossible places (playing GIFs, QR codes).
5. **Keyboard-driven editing is Shottr's soul** (single-key tools, Tab hex, Enter crop) and what makes power users call it "fast." CleanShot's editor is mouse-first. A keyboard-first editor with CleanShot polish would be "best of both worlds" literally as reviewers define it.
6. **Recording is CleanShot's #1 retention feature for pros** (Loom-replacement + internal audio + pause/restart + link-before-upload-finishes) — multiple reviewers say it's the feature they use "dozens of times a day." Deferring it to v2 is defensible for v1 scope, but the video evidence says it's the single biggest reason people pay and stay. Catalog: webcam PiP swap mid-recording, mouse-click/keystroke overlays, auto-DND, FPS control, GIF export, trim editor with live file size.
7. **Cloud is polarizing:** recipients and client-facing users love it (custom domain, self-destruct, password); prosumers actively coach skipping the subscription and using Google Drive. A local-first v1 matches a real, vocal segment — but instant-link sharing is the loudest "wow" for the client-communication persona.
8. **Pricing feel:** one-time $29 is praised as fair AND named as "a hard pill"; the $8–10/mo Cloud Pro is the trust-breaker; Shottr's free-with-nag → $8 is described as underpriced by reviewers. No free trial for CleanShot is a recurring purchase-anxiety moment (Setapp is the de-facto trial). Evidence points to: generous free/trial tier, honest one-time core price, subscription only for genuinely recurring cost (cloud/AI).
9. **Utility hooks beyond capture create daily retention:** hide-desktop-icons during Zoom calls, pin-to-screen reference windows, hex picking, pixel rulers — reviewers use these "when not even screenshotting." These are cheap, lovable v1 candidates.
10. **The comparison frame in the market is already "CleanShot vs Shottr vs Screen Studio (vs Snagit)"** — even AI-generated channels produce it. Positioning language that already exists in viewers' heads: CleanShot = "versatile all-in-one," Shottr = "lightning fast, lightweight, pixel-perfect, free," Screen Studio = "polished video." The unclaimed slot in reviewer vocabulary: *fast like Shottr, beautiful like CleanShot, honest pricing.*
