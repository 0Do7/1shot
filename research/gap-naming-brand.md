## 2. Naming conventions & saturation

### Saturated roots ([OBSERVED] counts from §1 inventory, 2026-06-09)
- **`shot`** — ~25+ live products (CleanShot, Shottr, Lightshot, Greenshot, Flameshot, Longshot, iShot, Better Shot, Tidyshot, ShotX, Shotter, Shotty, FinalShot, UltimateShot, NeoShot, McShot, Madshot, Shotium, ShotFit, Shoots, StoreShots, SafeShot, ShotSolve, AutoShot, Shotomatic…). **Most saturated root in the category.**
- **`snap`** — ~15+ (Snagit-adjacent, Xnapper, Monosnap, SnapNDrag, RapidSnap, QuickSnapX, CleanSnapX, SnapPro, Snap Plus, Snabbit, ScreenSnapAI, ScreenSnap Pro, Key Snap, WebSnap…). Also collides with Snapchat/Snap Inc. mindshare [analysis].
- **`snip`** — ~10 and genericized by Microsoft's "Snipping Tool" (Xnip, Snipaste, Snip (Tencent), Snip My, Snipit, Snipin, SnipWeb, TextSniper…).
- **`screen-` prefix** — endless, descriptive, SEO-contested (and now being used precisely *as* SEO bait: screensnap.pro, shotomatic.com blogs).
- **`capture/cap`** — moderately used (Capto, Capso, Cap, Scapture, 1Capture, PolyCapture, Capuchin).
- **`X` suffix/prefix** — CleanShot X, ShareX, Xnip, Xnapper, ShotX, QuickSnapX, CleanSnapX — and clone farms gravitate to X-names (§3), so an X-name now pattern-matches to the clones [analysis].
- **Vowel-drop `-r/-lr`** (Shottr, Droplr, prntscr) — dated Web-2.0 pattern with demonstrated misspelling/squat cost ("Shotter" app, §3).

### Names that encode the value prop (the CleanShot playbook)
- **CleanShot** = clean (icon-free) shot — founders' origin story [CLAIM, nocsdegree.com].
- **PixelSnap** (same founders, MakeTheWeb) = snap pixel measurements.
- **Snipaste** = snip → paste/pin. **Longshot** = long scrolling shot. **ScreenFloat** = floating shots. **TextSniper** = snipe text (OCR). **Screenotate** = screenshot + annotate. **Lightshot** = lightweight. **Shotomatic** = automatic shots. **Tidyshot** = tidy library. **Snagit** = snag it (speed/grab). **Better Shot** = comparative claim. **Skrin/Zight** = respelled generic ("screen"/"sight").
- Pattern: winners encode **one** hero benefit, in plain words, ≤3 syllables [analysis].

### Unclaimed semantic territories ([OBSERVED] absence in inventory, 2026-06-09)
- **Memory / library / recall / "your screenshots are a knowledge base"** — only micro-apps (Eidetic, camp, Knud, Pickle, Tidyshot) sit here; none has brand gravity. This matches your "library/search" positioning slot.
- **Local-first / private / yours** — heavily used in *copy* (Pickle: "No accounts. No tracking."; Better Shot: "No account, no cloud, no subscription"; Capso: "no telemetry") but **no one encodes it in a name**.
- **Speed** — claimed in copy everywhere ("blazingly fast", "in 2 seconds") but in names only weakly (RapidSnap, a dead 2023 freebie).
- **Pixels / precision** — Shottr owns the audience ("those who care about pixels") but not the word; Pixera is squatting nearby with a collision-laden name.
- **Still / frame / exposure (photography metaphors)** — "frame" names exist only in the App-Store-marketing-mockup subcategory (Framely, FrameBuddy, ShotFit, Frame Screenshots); "still", "develop", "darkroom"-style territory untouched in *this* category (Darkroom is a photo editor — collision to check).
- **Clean/beauty** — owned by CleanShot and chased by the entire 2024–26 beautifier wave (Xnapper, Pixera, Skrin, Better Shot); crowded in copy, claimed in name only by CleanShot.

## 4. How users say the names (radio test)

Direct caption/misspelling VOC was thin in indexable sources; structural evidence is stronger:

- **[OBSERVED]** "Shotter: Screenshot & Recorder" exists on MAS (2026-04-24) — someone is already monetizing Shottr's phonetic spelling. The misspelling is common enough to squat.
- **[OBSERVED]** "iScreen Shoter" — a second spelling-instability squat in the same results.
- **[CLAIM — project's prior research, not independently verified 2026-06-09]** YouTube auto-captions render "CleanShot X" as "clean shot axe" and "Shottr" as "shotter/shatter"; Reddit shorthand "cleanshot", "CSX". My searches could not surface these verbatim (Reddit largely unindexed for these queries); recommend manual YouTube-transcript spot-check of 3–5 review videos.
- **[OBSERVED]** Community shorthand drops suffixes: HN commenters say "Cleanshot" (lowercase, no X) — VOC: "if you want to pay, get Cleanshot" (gingerlime, https://news.ycombinator.com/item?id=31773863, 2022 ⚠️stale); "Cleanshot is amazing" (steve-benjamins, same thread). The "X" does not survive word-of-mouth.
- **[OBSERVED]** Zight's stated rebrand rationale is a canonical radio/search-test failure writeup: "people confused it with a plethora of other cloud storage solutions from iCloud to Google Cloud, and even an application to identify clouds… securing and defending a generic name like 'CloudApp' was an ongoing legal and SEO challenge" (zight.com/blog/cloudapp-is-now-zight, 2023 ⚠️stale).
- Radio-test failure modes represented in the category: vowel-drop (Shottr→shotter), letter-suffix (X→"axe"/dropped), ambiguous initial (Xnapper: ex-napper vs snapper), respelled generic (Skrin→screen, Zight→sight/zite) [analysis on observed names].

## 6. Setapp & rename precedents

- **Setapp**: CleanShot X and Xnapper are both distributed on Setapp under their full names (setapp.com/apps/cleanshot, setapp.com/apps/xnapper, [OBSERVED] 2026-06-09). Setapp's own editorial calls CleanShot X "the most popular Mac snipping tool" [CLAIM, setapp.com/how-to/snipping-tool-for-mac, 2026]. Setapp's developer requirements ([OBSERVED] https://docs.setapp.com/docs/preparing-your-application-for-setapp, 2026-06-09) impose **no app-name rules** beyond "App description and other text metadata must be relevant to the app content and functionality"; they do ban in-app purchases/donations and demo/trial builds in the Setapp version. No evidence of a category name dispute on Setapp. Practical notes: naming is effectively unconstrained by Setapp; listing pages live at /apps/<name> so short, unique slugs win; and Setapp distribution puts a new entrant's name directly beside CleanShot X's flagship listing on the same storefront.
- **CloudApp → Zight (2023 ⚠️stale)** — the category's only documented rename:
  - Rationale [OBSERVED, zight.com/blog/cloudapp-is-now-zight]: generic name caused (a) user confusion with iCloud/Google Cloud "and even an application to identify clouds", (b) "an ongoing legal and SEO challenge" to defend, (c) enterprise-procurement friction — VOC (company): "telling IT departments to install 'CloudApp' never went well"; replacement chosen because "Zight is distinctive, trademarkable, and ownable… a playful variation rooted in the word 'sight'".
  - Reception: mixed-to-negative in reviews, mostly via *conflation with product decline*: VOC: "Used it daily when it was still Cloudapp, now with new branding the software unfortunately is buggy and there are no updates. Looks abandoned." ([CLAIM] review via softwareadvice.com/customer-success/cloudapp-profile/reviews, 2024). Review platforms still index it as "Zight (formerly CloudApp)" three years on (capterra.com/p/154298/CloudApp), i.e., the old name's equity never fully transferred. The project's prior "rename poorly received" finding is directionally supported but the loudest verbatim complaints were not independently re-found in this pass.
  - Lesson pair: CloudApp shows the cost of a *generic* name; Zight shows the cost of changing it later. Skitch shows a loved name can outlive its product by a decade (still the reference point in 2022 HN threads and 2025 blog posts, masutaka.net/en/2025-10-22-1).
